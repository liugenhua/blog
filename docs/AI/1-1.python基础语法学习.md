## 2. 变量作用域
你已经学会：

+ 局部变量：定义在函数内部，只能在函数内部使用
+ 全局变量：定义在函数外部，整个文件中通常可用
+ 函数里可以读取全局变量
+ 函数里如果要修改全局变量，要用 global
+ 函数参数本质上也是局部变量

重点结论：

+ 函数内定义的变量，默认是局部变量
+ 函数外定义的变量，默认是全局变量
+ 修改全局变量时要写 global

## 6. 继承、重写、super、多态
你已经学会：

+ 继承：子类可以直接获得父类的属性和方法
+ 方法重写：子类写同名方法，覆盖父类方法
+ super()：在子类中调用父类的方法或 __init__
+ 多态：同样的方法，不同对象调用时有不同表现

重点结论：

+ 继承是复用代码
+ 重写是改写父类行为
+ super().__init__(...) 常用于继承父类属性
+ 多态的核心是“同样调用，不同结果”

## 11. 路径和 os 基础
**1. ****os**** 基础**  
认识了 os 模块的作用：用来和操作系统交互，常用于目录、文件、路径相关操作。

重点内容：

+ os.getcwd()：获取当前工作目录
+ os.listdir()：列出当前目录下的文件和文件夹
+ os.mkdir()：创建文件夹
+ os.remove()：删除文件
+ os.rmdir()：删除空文件夹
+ os.chdir()：切换工作目录

理解重点：

+ os 偏向“操作系统动作”
+ 经常和文件、目录、路径一起使用

**2. ****os.path**** 路径处理**  
学习了 os.path 主要用于处理路径。

重点内容：

+ os.path.exists(path)：判断路径是否存在
+ os.path.isfile(path)：判断是不是文件
+ os.path.isdir(path)：判断是不是文件夹
+ os.path.join(a, b)：拼接路径
+ os.path.abspath(path)：获取绝对路径
+ os.path.basename(path)：获取最后一部分，一般是文件名
+ os.path.dirname(path)：获取目录部分
+ os.path.splitext(path)：分离文件名和扩展名

理解重点：

+ exists() 只判断存不存在
+ isfile() 和 isdir() 是进一步判断类型
+ join() 比手动拼接字符串更规范

**3.**** ****pathlib**** ****基础**  
知道了 pathlib 是更现代的路径处理方式，比 os.path 更清晰。

重点内容：

+ from pathlib import Path

+ p.exists()：判断路径是否存在

+ p.stem：主文件名

理解重点：

+ pathlib 也是处理路径
+ 和 os.path 相比更面向对象、更现代

## 13. requests 和 API 调用
  
开始系统理解 Python 如何请求接口。

重点内容：

+ requests.get(url)：发送 GET 请求
+ requests.post(url)：发送 POST 请求
+ response.status_code：查看状态码
+ response.text：查看文本内容
+ response.json()：把返回的 JSON 转成 Python 对象
+ params=：给 GET 请求传参数
+ json=：给 POST 请求传 JSON 数据
+ timeout=：设置超时时间
+ raise_for_status()：状态码异常时主动报错

理解重点：

+ GET 常用于获取数据
+ POST 常用于提交数据
+ 调接口的本质是：发送请求 -> 拿到返回 -> 解析 JSON -> 取值

### 4、环境相关关键理解
今天比较重要的一个收获是：

_全局环境 和 虚拟环境 是两套独立环境。_

具体理解为：

+ 全局 Python 里装的包，不等于 .venv 里也有
+ .venv 里安装的包，也不会自动跑到全局环境
+ 进入虚拟环境后，优先使用：`<font style="background-color:rgb(43, 43, 43);">python -m pip  
</font>`

而不是只盯着 py -m pip。

## 17、异常处理基础
今天学习了用 try/except 包住请求代码：

```python
try:
response = requests.get(url, timeout=5)
data = response.json()
except requests.exceptions.RequestException as e:
print("请求出错了：", e)
except ValueError as e:
print("JSON 解析出错了：", e)
except Exception as e:
print("其他错误：", e)
```

当前理解：

+ RequestException：处理请求相关问题
+ ValueError：处理 JSON 解析问题
+ Exception：做最后兜底
